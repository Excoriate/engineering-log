# Top patches required

1. **Relocate the DNA block or add a navigation anchor.** SKILL.md lines 33-93 (~1800 tokens) fire before the Skill Selection Gate, the Input-Mode Decision, and all heuristics. Per Law 2 (Recency Dominance), the agent's attention peak is at the END of the spine — the DNA axioms at the top will be under-attended when the agent is drafting levels 6-12. The DNA section currently reads like a manifesto; it must become *the opening of each phase gate* or be condensed to a 5-line summary at the top with `→ see DNA block at end` and the full text moved to the bottom half of SKILL.md. Without this move, the DNA will be context-skimmed on every real invocation. **Impact if not patched**: agents will cite the DNA in phase 0 then fall back to template-filling by phase 4 — exactly the drift Q8 describes.

2. **Collapse tests 3 and 4 in mental-model-construction.md and make the minimum explicit.** Tests 3 (predictive) and 4 (transferable) are not discriminating — any model that passes test 3 necessarily passes test 4 (if a model generates a falsifiable prediction about a *different* incident, stripping incident-nouns trivially leaves a useful statement). The reverse also holds: a model that passes test 4 (strip-and-survive) implies predictive power. Retaining both imposes redundant work without adding discrimination. The true minimum is {named, compressed, predictive/transferable, connected} = 4 tests. Collapse tests 3+4 into one: "Is it predictive AND transferable? State the prediction AND show the noun-stripped residue." **Impact if not patched**: agents will apply five identical-outcome tests and experience false confidence that the model is fully validated.

3. **Add a graceful-degradation path for domain-unfamiliar invocations in SKILL.md.** The skill implicitly assumes the agent holds prior knowledge of the system (it must name the L1 role-metaphor, connect L10 lessons to sibling RCAs, fill the model-graph). When the agent doesn't know the system, there is no honest failure path — the skill's own discipline (test 5: connected; test 3: predictive) will fail silently, and the agent will hallucinate connections and predictions rather than surface uncertainty. Add a Phase 0 check: "If I cannot independently verify ≥2 load-bearing system claims from this session's evidence, I am in SYNTHESIZE mode with LOW domain prior. I must flag every model-construction step 4 (axiom) and step 5 (prediction) as [UNVERIFIED[assumption: domain-prior insufficient; resolving probe: <named probe>]]." **Impact if not patched**: the skill actively encourages confident-looking hallucinated models on unfamiliar domains — the worst possible failure mode for an epistemic discipline tool.

**Q3 — Four DNA axes: are they orthogonal?**
Working example: "the runtime CG `activation-mfrr` is missing on `dispatcher-output-1`."

- **Axis 1 (Angles)**: Structural angle — the resource topology has a gap (CG missing from the EH). Behavioral angle — the pod startup sequence hits two parallel `ResourceNotFound` / `ContainerNotFound` exceptions. Failure-modal angle — IaC-declared-but-unapplied is the failure class; investigate delivery layer. These three are genuinely distinct outputs.
- **Axis 2 (Perspectives)**: Next-shift on-call — "which probe confirms the gap?" (az CG list). Operator — "which pipeline do I trigger and approve?" (terraform-cd-sandbox). Developer — "which yaml line is the leverage point?" (trigger: none → push: branches). Future on-call — "what's the 5-minute recognition path?" (CG missing + pipeline skipped). These are distinct outputs.
- **Axis 3 (First principles)**: Universal principle: "when IaC declares a resource and runtime lacks it, the gap is in the delivery layer; investigate trigger: and approval gates before re-editing the declaration." This is distinct from axes 1 and 2 — it is the axiom, not the observation.
- **Axis 4 (Systems thinking)**: Interactions — merge event → pipeline → approval gate → CG existence. Boundaries — IaC boundary vs runtime boundary. Time — 5-day latent drift between merge and surfacing.

**Verdict: the four axes ARE orthogonal for this example.** Each produces content the others don't. The concern is semantic coupling between Axis 1 (structural angle) and Axis 4 (interactions/boundaries) — both deal with components and edges. An agent that conflates them will produce two nearly-identical lists. The SKILL.md DNA section (lines 41-82) partially addresses this by mapping each axis to different levels, but the distinction between "structural angle" (what components exist, how they connect) and "systems thinking" (interactions, boundaries, time) is under-specified. Axis 1 sees *what*; Axis 4 sees *how they relate over time*. This distinction should be one clarifying sentence in the DNA block. Without it, Law 3 (Semantic Priming) will cause "structural" and "systems" to activate the same training associations.

**Q5 — Voice consistency.**

**Q6 — Model-graph as enforcement mechanism.**
mental-model-construction.md lines 68-104 shows the 12-model dependency graph. Validator G11 (vocabulary continuity, SKILL.md line 183) catches surface-level name reuse but does NOT catch model-dependency violations: an agent could produce a correct vocabulary-continuity chain while installing models in the wrong order or skipping a dependency. Example: if L6's delivery-state-machine model is weak or missing, L8's minimum-leverage-change model becomes undefendable — but G11 will not flag this because L8 may still reuse a vocabulary term from L6. **What G11 misses**: (a) a level that names a vocabulary term but does not install the underlying model; (b) a model at level N that depends on the model at level N-2 (transitive dependency) but level N-1 was skipped with a non-null vocabulary term. The model-graph is aspirational unless Phase 3 explicitly requires the agent to verify each level's named model candidate against the graph's dependency structure. Current Phase 3 gate does not require this.

**Q9 — Epistemic capacity and domain-unfamiliarity.**
The skill requires at Phase 4: (a) name the model (requires domain knowledge to make metaphors), (b) test for transferability (requires knowing other incidents in the same class), (c) connect to existing knowledge (requires knowing sibling RCAs and ADRs), (d) write a narrative arc with epistemic passion (requires having actually felt the surprise). None of these are gracefully degradable in the current skill text. The closest the skill comes to acknowledging this is SKILL.md line 304-305: "if neither socrates-contrarian nor el-demoledor is deployed, run the questions yourself and downgrade the result." But domain-unfamiliarity is different from missing reviewers — the agent may have reviewers but no ground truth to work from. **Failure mode**: an agent on an unfamiliar domain will generate plausible-sounding role-metaphors (L1), plausible-sounding axioms (Axis 3), and plausible-sounding sibling-RCA connections (test 5) — all of which are untethered to evidence. The skill's own evidence-classification discipline (A1/A2/A3) would flag these as INFER or UNVERIFIED, but the mental-model-construction protocol does not explicitly require A-labeling on model-construction steps 4 and 5. **Required fix**: see Patch 3 above.
